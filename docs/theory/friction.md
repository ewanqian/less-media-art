# 摩擦 (Friction)

> "顺畅不是美德，阻力才是创造力的朋友。"

## 核心定义

**摩擦**是指在数字创作过程中**主动引入的阻力、延迟或不顺畅**。在少媒体艺术的语境下，摩擦不是需要消除的bug，而是**需要设计和珍视的特征**。

现代数字工具追求"无缝体验"——零延迟、零阻力、零学习成本。但少媒体艺术认为，这种"顺畅"是有代价的：它让我们成为被动的消费者，而非主动的创作者。

## 为什么需要摩擦？

### 1. 抵抗即时满足

AI工具可以在毫秒级响应。这种即时性：
- 消除了思考的延迟
- 鼓励快速消费而非深度创作
- 让我们期待"答案"而非享受"探索"

**摩擦通过引入延迟，迫使我们等待和思考。**

### 2. 重获身体感知

数字创作往往是"无身体"的——手指轻触屏幕，思维直接变成输出。

摩擦重新引入**身体性**：
- 等待时的焦虑
- 限制时的挣扎
- 失败时的沮丧

这些"不舒服"的感受提醒我们：**创作是身体与物质的交互，不是纯粹的精神活动**。

### 3. 显形选择的价值

当一切都很顺畅时，我们不会意识到自己在做选择。

摩擦让每个选择都变得**可见和沉重**：
- 为什么要等待5分钟才能继续？
- 为什么要手动删除前一句话？
- 为什么要限制token数量？

这些"不必要的麻烦"迫使我们追问：**我真的需要这个吗？**

## 摩擦的类型

### 时间摩擦

**定义**: 引入时间延迟

**示例**:
- 每次生成后必须等待5分钟
- 冷却期随轮次递增（第1轮1分钟，第2轮2分钟...）
- 固定总时长（如30分钟必须结束）

**效果**:
- 培养耐心
- 让"等待"成为创作的一部分
- 接受"未完成"的状态

### 界面摩擦

**定义**: 使用不那么"友好"的界面

**示例**:
- 命令行而非图形界面
- 纯文本而非富文本
- 没有自动保存

**效果**:
- 每次输入都需要完整思考
- 错误有真实代价
- 技术变得可见

### 认知摩擦

**定义**: 引入需要额外认知努力的任务

**示例**:
- 必须手动记录日志
- 必须解释每个决策的理由
- 必须为每个输出提供反向论证

**效果**:
- 显形通常自动化的思维过程
- 培养元认知能力
- 创造反思的空间

### 物质摩擦

**定义**: 让硬件限制变得可感知

**示例**:
- 使用CPU而非GPU（更慢）
- 限制内存使用
- 在旧电脑上运行

**效果**:
- 听到风扇转动，感知计算成本
- 体验"慢"的美学
- 理解技术条件如何塑造创作

## 在少媒体艺术中的实践

### 案例：冷却期模式

```
第1轮生成 → 等待1分钟 → 第2轮生成 → 等待2分钟 → 第3轮生成 → 等待3分钟...
```

**记录要点**:
- 等待时的想法
- 是否试图绕过限制
- 延迟如何改变了第N轮的输入

### 案例：离线模式

```
1. 提前下载模型
2. 关闭WiFi
3. 记录硬件状态（温度、风扇速度）
4. 创作
5. 重新联网，提交案例
```

**记录要点**:
- 离线时的感受（焦虑？解放？）
- 硬件限制如何影响创作
- 与在线创作的对比

## 批评与回应

**批评**: 摩擦是否只是苦行主义？为了难受而难受？

**回应**:
- 确实存在这种风险
- 关键在于**意图**：摩擦是为了显形和反思，不是为了受苦
- 如果摩擦没有带来洞察，它就是无意义的

**批评**: 在效率至上的社会，摩擦是否可行？

**回应**:
- 这正是为什么需要摩擦——**抵抗效率霸权**
- 少媒体艺术不主张所有创作都应该有摩擦
- 但主张：**至少有些创作应该有摩擦**，作为反思和抵抗的空间
