# 过程显形 (Process Visibilization)

> "过程不是通向作品的手段，过程本身就是作品。"

## 核心定义

**过程显形**是指将通常被隐藏、被忽视或被刻意抹除的创作过程，**作为作品的核心组成部分加以呈现**的实践。在AI创作的语境下，这意味着：

- 展示与AI的完整对话历史
- 记录决策的犹豫与回溯
- 保留错误、失败和"不走运"的尝试
- 显形技术限制如何塑造了结果

## 为什么需要过程显形？

### 1. 对抗AI的"黑箱"

生成式AI的输出往往是"魔法般"的即时结果：输入提示词，输出图像/文本。这种**无缝性**遮蔽了：
- 模型内部的决策机制
- 训练数据的来源与偏见
- 生成过程中的概率选择

过程显形将创作重新**事件化**，让观众/读者看到：这个结果不是"凭空出现"的，而是**一系列具体决策的产物**。

### 2. 重获作者性

当AI可以生成"完美"的结果时，**选择展示什么**成为人类创作者的新权力：
- 选择记录哪一轮对话
- 选择保留哪些"失败"尝试
- 选择如何编排过程的叙事

### 3. 理论价值

过程记录成为**研究材料**：
- 分析创作思维的路径
- 比较不同创作者的决策模式
- 理解约束如何影响过程

## 过程显形的层次

### 层次1：原始日志 (Raw Logs)

最基础的记录：
```
[2025-02-02 10:23] 用户：请写一首关于春天的诗
[2025-02-02 10:23] AI：春风拂面柳丝长...
[2025-02-02 10:24] 用户：太普通了，禁止用"春"字
[2025-02-02 10:24] AI：东风解冻，万物复苏...
```

### 层次2：注释日志 (Annotated Logs)

加入创作者的反思：
```
[10:23] 提示词："请写一首关于春天的诗"
    ↳ 反思：这个提示太宽泛了，AI给出了陈词滥调
    ↳ 决策：施加约束，禁止用"春"字

[10:24] 提示词："禁止用'春'字，重写"
    ↳ 反思：约束迫使AI使用更间接的意象
    ↳ 观察："东风解冻"比"春风拂面"更有质感
```

### 层次3：过程叙事 (Process Narrative)

将日志组织成有结构的故事：
- **起点**：初始意图与预期
- **转折**：遇到的阻力与意外
- **决策**：关键的选择时刻
- **终点**：为何停止？满意还是妥协？

## 在少媒体艺术中的实践

### 案例提交规范

每个案例必须包含：
1. **元数据**：时间、工具、约束条件、硬件环境
2. **原始日志**：完整的交互记录
3. **创作者陈述**：意图、反思、关键转折点
4. **开放批注**：允许他人添加评论和分析

### 工具支持

`constraint-cli` 自动记录：
- 时间戳
- 约束条件
- 完整对话
- 系统资源使用（CPU、内存）

## 批评与回应

**批评**：过程显形是否只是一种"真诚表演"？展示过程是否成为新的"风格"？

**回应**：
- 真诚的问题确实存在，无法完全避免
- 但**有记录的过程**比**无记录的黑箱**更易于批判性审视
- 框架鼓励对"过程显形"本身的反思：你为什么选择展示这个过程？
