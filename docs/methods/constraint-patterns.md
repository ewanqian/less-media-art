# 约束设计模式库

这是一个由社区贡献的"约束"案例集合。每个模式都包含：**规则描述**、**设计意图**、**预期效果**、**可能的讨论**。

## 如何使用

1. **选择模式**：浏览以下模式，选择适合你当前创作目标的
2. **记录过程**：使用 `constraint-cli` 或其他工具记录完整过程
3. **提交案例**：将过程日志和反思提交到案例档案

## 语法约束

### 模式1: Token限制

**规则**: 仅使用模型输出的前N个token（如100个）

**设计意图**: 
- 迫使模型在有限空间内表达
- 创造"未完成"的美学
- 显形语言的压缩与省略

**预期效果**:
- 回答更简洁、更密集
- 出现意外的省略和跳跃
- 需要更多轮次来展开想法

**可能的讨论**:
- 100个token是否是最优限制？
- 这种限制是否只是制造了"残缺"而非"精炼"？

**示例**:
```
约束: 仅使用100个token
提示: 解释量子力学

输出: [截断在100token处，可能停在句子中间]
```

---

### 模式2: 禁用词汇

**规则**: 禁止使用特定词汇或词性（如所有形容词）

**设计意图**:
- 打破语言习惯
- 迫使寻找新的表达方式
- 探索语言的边界

**变体**:
- 禁用"的"（中文）
- 禁用"the"（英文）
- 禁用所有情感词汇
- 禁用第一人称

**示例**:
```
约束: 禁用所有形容词
提示: 描述一个悲伤的场景

输出: 雨落在街道上。一个人坐在窗边。茶杯变凉了。
```

---

### 模式3: 格式强制

**规则**: 输出必须符合特定格式（如诗歌、代码注释、JSON）

**设计意图**:
- 形式即内容
- 让形式约束生成新的想法
- 探索不同文体的思维方式

**变体**:
- 必须是俳句（5-7-5）
- 必须是Python代码注释
- 必须是对话体
- 必须是清单体

## 逻辑约束

### 模式4: 单一主题

**规则**: 整个对话只能围绕一个核心概念展开，禁止引入新概念

**设计意图**:
- 深入挖掘一个想法的所有维度
- 抵抗思维的跳跃和分散
- 练习"停留"的能力

**示例**:
```
主题: 水
允许: 水的物理性质、象征意义、历史角色...
禁止: 突然开始讨论火或空气
```

---

### 模式5: 递归回应

**规则**: 每一轮回应必须包含并重构前一轮的所有论点

**设计意图**:
- 显形思维的累积和演变
- 防止"遗忘"和"跳跃"
- 创造层叠的叙事结构

**示例**:
```
第1轮: A
第2轮: A + B (B是对A的发展)
第3轮: A + B + C (C是对A和B的发展)
...
```

---

### 模式6: 反向论证

**规则**: 必须为每一个生成的观点提供反向论证

**设计意图**:
- 抵抗AI的"确认偏见"
- 培养批判性思维
- 显形思想的复杂性

**示例**:
```
AI: 这个观点是X，因为...
但也存在反对意见：...
```

## 系统约束

### 模式7: 离线模式

**规则**: 断开网络连接，仅使用本地资源

**设计意图**:
- 体验真正的"本地性"
- 感受硬件的物质限制
- 抵抗云端的"无限"幻觉

**技术要求**:
- 提前下载所需模型
- 关闭WiFi/网络
- 记录硬件状态（温度、风扇速度）

---

### 模式8: 资源限制

**规则**: 限制计算资源（如仅使用CPU，或限制内存使用）

**设计意图**:
- 显形计算的物质成本
- 体验"慢"的价值
- 理解技术条件如何塑造结果

**实现方式**:
```bash
# 限制CPU核心数
taskset -c 0 ollama run llama3.2:3b

# 或限制内存
cgexec -g memory:limit1G ollama run llama3.2:3b
```

---

### 模式9: 模型降级

**规则**: 使用更小、更"弱"的模型

**设计意图**:
- 体验"能力不足"带来的创造力
- 抵抗"更大更好"的技术叙事
- 发现小模型的独特"美学"

**示例**:
```
正常: llama3.2:70b (高质量但慢)
约束: llama3.2:1b (低质量但快且有"瑕疵")
```

## 时间约束

### 模式10: 冷却期

**规则**: 每次生成后必须等待固定时间（如5分钟）才能继续

**设计意图**:
- 抵抗即时满足
- 培养耐心和深思熟虑
- 让"等待"成为创作的一部分

**变体**:
- 线性增长（第1轮等1分钟，第2轮等2分钟...）
- 随机延迟（随机等待1-10分钟）

---

### 模式11: 限时创作

**规则**: 整个创作过程必须在固定时间内完成（如30分钟）

**设计意图**:
- 创造紧迫感
- 接受"未完成"的状态
- 记录时间压力下的决策

---

## 组合模式

### 模式12: 极简主义套装

**组合**:
- Token限制: 50个
- 禁用: 所有形容词
- 格式: 俳句
- 模型: 最小的可用模型

**设计意图**: 极端约束下的创造力实验

---

### 模式13: 深度挖掘套装

**组合**:
- 单一主题
- 递归回应
- 冷却期: 每轮2分钟
- 日志记录: 详细注释

**设计意图**: 对一个概念的深度探索
