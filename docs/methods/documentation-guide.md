# 文档规范指南

本文档规定了少媒体艺术框架中所有文档的编写标准。

## 文件格式

所有文档使用 **Markdown** 格式，文件扩展名为 `.md`。

## 文件头（Front Matter）

每个文档文件必须包含YAML格式的文件头：

```yaml
---
title: "文档标题"
description: "简短描述（50字以内）"
---
```

### 可选字段

对于案例档案，还可以包含：

```yaml
---
title: "案例标题"
author: "作者名"
date: "YYYY-MM-DD"
type: "socratic_dialogue | constraint | locality"
topic: "主题关键词"
---
```

## 文档结构

### 理论词条结构

```markdown
# 标题

> 引用/核心观点

## 核心定义

[概念的定义]

## 为什么重要？

[解释这个概念的价值]

## 具体内容

[分节详细阐述]

## 批评与回应

[预判并回应可能的批评]

## 延伸阅读

- [书名/文章名](链接)

## 相关词条

- [相关概念](链接)
```

### 方法指南结构

```markdown
# 标题

## 目标

[需要什么准备]

## 步骤

1. 第一步

[具体例子]

## 常见问题

### 问题1？
答案...

- [理论词条](链接)
```

## 写作风格

### 语言

- 使用简洁、直接的中文
- 避免过度学术化的术语
- 必要时提供英文对照

### 语气

- 邀请式而非命令式
- 反思性而非断言式
- 开放性而非封闭性

### 示例

❌ 不要："用户必须遵守以下规则..."
✅ 要："我们邀请你尝试以下实践..."

❌ 不要："这就是正确的做法..."
✅ 要："这种方法可能带来以下效果..."

## 链接规范

### 内部链接

使用相对路径：

```markdown
[媒介减法](../theory/media-subtraction.md)
[环境搭建](environment-setup.md)
```

### 外部链接

使用完整URL：

```markdown
[Lev Manovich](https://manovich.net)
```

## 图片规范

- 使用相对路径引用图片
- 图片放在 `assets/images/` 目录
- 提供替代文本（alt text）

```markdown
![图片描述](../assets/images/example.png)
```

## 代码块

使用 fenced code blocks，并指定语言：

```markdown
```python
def example():
    pass
```
```
